# Авито API

Минимальная заготовка проекта для работы с официальным API Авито без хранения секретов в коде.

## Что уже есть

- безопасное хранение ключей в `.env`
- PowerShell-скрипт для получения `access_token`
- PowerShell-скрипт для первой проверки Messenger API

## Как запустить

1. Скопируйте `.env.example` в `.env`
2. Заполните:
   - `AVITO_CLIENT_ID`
   - `AVITO_CLIENT_SECRET`
   - `AVITO_USER_ID`
3. Откройте PowerShell в папке проекта
4. Выполните:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Test-AvitoAuth.ps1
```

Если авторизация успешна, выполните:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Get-AvitoChats.ps1
```

## Что дальше

Если чаты читаются успешно, следующим шагом можно:

- получать новые сообщения
- отправлять автоответ с шаблонами
- добавить фильтр: отвечать только на первое сообщение в чате
- вынести тексты ответов в отдельный файл

## Важно

- Не отправляйте `AVITO_CLIENT_SECRET` в чат, почту и скриншоты.
- Если секрет случайно показали, перевыпустите ключи.
- Если API вернет ошибку по Messenger, это может означать, что для аккаунта или тарифа не хватает прав именно на сообщения.
